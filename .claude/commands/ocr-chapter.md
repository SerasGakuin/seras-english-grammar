# OCR章書き起こし

指定した章を書き起こしてMarkdownファイルとして保存する。

## 引数
- $ARGUMENTS: 書籍名 章番号（例: 核心 01、成川 Part05）

## 手順

1. **目次から章情報を取得**
   - `pdf/output/{書籍名}/00_目次.md` を読む
   - 指定章のページ範囲を特定

2. **画像準備**
   - 該当ページの画像が `pdf/images/{書籍名}/` にあるか確認
   - なければ、PDFから抽出

3. **書き起こし実行**
   - subagent (general-purpose) に以下を渡す:
     - 画像ファイルパス（章の全ページ）
     - 表記ルール（`.rules/notation-rules.md`）
     - 書籍名・章番号（コンテキスト）

4. **Codexレビュー**
   - `/ocr-review` を実行
   - 問題があれば修正

5. **スポットチェック判定**
   - 各書籍の最初の章 → 必ずチェック
   - 5章ごとに1章 → チェック
   - 該当する場合、オーケストレーターが画像と出力を比較

6. **保存**
   - `pdf/output/{書籍名}/{章番号}_{章名}.md` に保存
   - `progress/status.json` を更新

## subagentへの指示テンプレート

```
以下の画像ファイルを読み込んで、英語参考書の章をMarkdown形式で書き起こしてください。

書籍: {書籍名}
章: {章番号} {章名}

画像ファイル:
{画像ファイルパスのリスト}

表記ルール:
{.rules/notation-rules.md の内容}

ルール:
1. 本文、例文、解説を正確に書き起こす
2. 図表は [図表: 説明] の形式で記載
3. ページ番号は各ページの区切りとして記載
4. 余計な説明は不要、本文のみ出力

出力形式: Markdownテキストのみ
```

## 注意事項
- 1章ずつ処理する（複数章を一度に処理しない）
- オーケストレーター自身は画像を読まない（subagentに委譲）
- 新しい表現が出てきたら、表記ルール更新を検討
