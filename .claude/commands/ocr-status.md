# OCR進捗確認

全体の進捗状況を表示する。

## 引数
- $ARGUMENTS: なし（または書籍名を指定して詳細表示）

## 手順

1. **進捗ファイル読み込み**
   - `progress/status.json` を読む

2. **全体サマリー表示**
   ```
   ## OCR進捗状況

   | 書籍 | ステータス | 完了章 | 残り |
   |------|-----------|--------|------|
   | 核心 | 目次完了 | 0/21 | 21 |
   | 成川 | 目次完了 | 0/127 | 127 |
   | ... | ... | ... | ... |
   ```

3. **書籍指定時は詳細表示**
   - 各章の完了状況
   - 最終更新日時
   - 次に処理すべき章

4. **次のアクション提案**
   - 未着手の書籍があれば提案
   - 処理中の書籍の次章を提案

## 出力フォーマット

```markdown
## OCR進捗状況（{現在日時}）

### 全体
- 完了: X/6冊
- 処理中: X冊
- 未着手: X冊

### 書籍別

| 書籍 | ステータス | 進捗 | 最終更新 |
|------|-----------|------|---------|
| 核心 | 章処理中 | 5/21章 | 2026-02-01 |
| ... | ... | ... | ... |

### 次のアクション
- `/ocr-chapter 核心 06` で次章を処理
```

## ドキュメント整合性チェック

進捗表示の最後に、ドキュメント整合性をチェックする：

```bash
.venv/bin/python scripts/sync_docs.py --check
```

- OK: 「✅ ドキュメントは最新です」と表示
- NG: 「⚠️ ドキュメント更新が必要です」と表示し、同期コマンドを案内

## 注意事項
- 進捗ファイルが存在しない場合は初期化を提案
- 不整合がある場合は警告を表示
- ドキュメント不整合時は `python scripts/sync_docs.py` を案内
